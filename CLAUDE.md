# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What This Is

NAV AiDE is an offline-first London travel assistant MVP for international tourists. React Native bare workflow (no Expo). Core features must work in airplane mode.

## Commands

```bash
# Install
npm install
cd ios && pod install && cd ..   # iOS only

# Build
npm run build                     # TypeScript compilation (tsc)

# Test (Node.js native test module via tsx)
npm test                          # runs tsx --test tests/**/*.test.ts

# Data pipeline (generate fixture assets + SQLite DBs)
npm run stage2:sql                # generate JSON + SQL scaffolds
npm run stage2:assemble           # assemble SQLite DBs from generated SQL
npm run stage2:validate           # validate DB tables, FTS, seed rows

# Full pipeline (assets + build + test)
npm run stage2:test

# React Native
npm run bundle:ios
npm run bundle:android
npm run ios                       # run on iOS simulator/device
npm run android                   # run on Android
npm start                         # Metro dev server

# Phase 3A verification
npm run phase3a:verify            # build + test + bundle both platforms
```

## Architecture

### Two-layer design

- **`src/core/`** — Pure offline business logic, testable in Node.js. No React Native dependencies.
- **`src/app/`** — React Native shell, mobile adapters, screens. Depends on core.

### Query pipeline (must preserve this sequence)

```
rawInput → FuzzyMatcher fast path → IntentExtractor → EntityResolver → POIService / Dijkstra → ValhallaBridge → ResponseRenderer
```

- **EntityResolver is mandatory before routing/search.** The LLM must not resolve locations.
- If confidence is below threshold, show disambiguation instead of guessing.
- The LLM (Gemma 4 E2B via `llama.rn`) is used **only** for structured intent extraction and natural-language response rendering. It must not do routing, POI lookup, or free-form reasoning.

### Key modules

| Area | Module | Purpose |
|------|--------|---------|
| Routing | `src/core/routing/Dijkstra.ts` | Shortest path on tube graph |
| Routing | `src/core/routing/ValhallaBridge.ts` | Walking routes via offline Valhalla tiles |
| Search | `src/core/poi/FuzzyMatcher.ts` | Fuzzy string matching |
| Search | `src/core/poi/POIService.ts` | SQLite FTS5 POI search |
| Pipeline | `src/core/pipeline/QueryPipeline.ts` | Main orchestrator |
| Pipeline | `src/core/pipeline/EntityResolver.ts` | Location/station name resolution |
| LLM | `src/core/llm/IntentExtractor.ts` | Structured intent extraction |
| LLM | `src/core/llm/ResponseRenderer.ts` | Natural language output |
| Contracts | `src/core/runtime/` | Runtime abstractions (model adapter, offline runtime, device ID) |
| Mobile | `src/app/pipeline/createMobilePipeline.ts` | Composes RN-safe QueryPipeline |
| Mobile | `src/app/screens/` | 4 screens: Go, Lost, Maps, Settings |
| Mobile | `src/app/state/AppShellContext.tsx` | Global app state via React Context |

### Data assets

- `assets/tubeGraph.json`, `assets/busRoutes.json` — deterministic London fixture graphs
- `assets/data/pois.db`, `assets/data/location_aliases.db` — SQLite DBs built from fixture SQL
- Generated by `scripts/data-pipeline/` scripts

### App screens (4-screen MVP)

GO | LOST? | Maps | Settings — bottom tab navigation via `@react-navigation/bottom-tabs`.

## Non-Negotiable Constraints

- **No cloud AI APIs.** All inference is on-device via `llama.rn`.
- **No Expo Go.** React Native bare workflow only.
- **No user accounts/login** in MVP.
- **No LiteRT-LM** in MVP.
- OS STT (`react-native-voice`) for speech input, OS TTS (`react-native-tts`) for speech output.
- All location grounding through local indices, never through LLM knowledge.
- Location names must remain in original English in every output.
- If LLM output contains a hallucinated place name, reject it.

## Testing

- Tests use Node.js native `test` module with `tsx` runner (no Jest/Mocha).
- **Unit tests** (`tests/unit/`): Dijkstra, EntityResolver, asset loaders, model bridge.
- **Golden tests** (`tests/golden/`): Integration tests with fixture-based assertions and hallucination checks. Test cases in `tests/golden/fixtures/query-pipeline-cases.json`.
- Core pipeline must be testable in Node.js before app integration.
- Changes to entity resolution, routing, prompt parsing, or output rendering require test updates.

## TypeScript

- Strict mode, target ES2022, JSX react-jsx.
- ES modules (`"type": "module"` in package.json).
- Node 20+ required.
